# 🚀 御宅追番 (Mikanani-Dog) 技术方案

## 💻 技术栈选择

### 后端方案

| 技术 | 方案选择 | 优势说明 |
|------|---------|---------|
| Web框架 | FastAPI | 高性能、易用的异步API框架 |
| 数据库 | SQLModel + SQLite | 轻量级、无需额外服务部署 |
| 任务处理 | 内置线程池 | 简化架构，满足基本需求 |
| 实时通信 | WebSocket | 提供下载进度实时更新 |
| 日志系统 | 结构化日志 | 方便调试和问题追踪 |

### 前端方案

| 技术 | 方案选择 | 优势说明 |
|------|---------|---------|
| 框架 | Vue 3 | 易用性高，社区成熟 |
| 状态管理 | Pinia | Vue生态集成良好 |
| UI库 | Naive UI + Tailwind CSS | 组件丰富，样式灵活 |
| 构建工具 | Vite | 开发体验好，构建速度快 |
| 实时通信 | Socket.IO | 与WebSocket配合使用 |

## 📋 核心功能

### 1. 番剧订阅

- **RSS管理**：添加、编辑、删除RSS订阅源
- **订阅规则**：支持关键词过滤、正则表达式匹配
- **更新检测**：定时检查RSS源更新

### 2. 资源管理

- **番剧解析**：自动解析番剧标题、集数信息
- **番剧规则**：支持手动编辑番剧解析规则
- **元数据获取**：对接TheTVDB/TMDb获取番剧信息

### 3. 下载功能

- **下载器支持**：对接qBittorrent、Transmission等
- **下载管理**：查看下载状态、暂停/恢复下载
- **下载通知**：下载开始/完成通知

### 4. 文件处理

- **智能重命名**：按模板自动重命名下载文件
- **自动分类**：按番剧名称整理到对应文件夹

### 5. 通知系统

- **多平台通知**：支持Telegram、Discord、微信等
- **事件通知**：新番更新、下载完成等事件通知

## 🛠️ 非功能需求

### 性能要求

- **响应时间**：API响应时间 < 300ms
- **内存占用**：基础运行 < 256MB

### 可用性要求

- **故障恢复**：自动重启机制
- **数据备份**：配置文件定期备份

### 安全要求

- **身份认证**：JWT认证
- **数据安全**：敏感配置加密存储
