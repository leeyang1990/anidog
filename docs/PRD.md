# 番剧自动下载管理系统 产品需求文档（PRD）

## 1. 产品概述

### 1.1 产品背景
随着番剧爱好者群体的不断扩大，手动追踪和下载番剧变得越来越繁琐。用户需要定期检查更新、手动下载、整理文件，这个过程耗时且容易遗漏。市场需要一个自动化、智能化的番剧管理解决方案。

### 1.2 产品定位
ANIDOG是一个基于Web的番剧自动下载管理系统，通过RSS订阅、智能过滤和自动化处理，为番剧爱好者提供一站式的追番解决方案。

### 1.3 产品目标
- **短期目标**（3个月）：实现核心功能，支持主流RSS源和基础的自动化下载
- **中期目标**（6个月）：完善用户体验，增加高级过滤规则和批量管理功能
- **长期目标**（12个月）：建立完整的媒体库管理生态，支持多种下载工具和存储方案

### 1.4 成功指标
- 用户平均每周节省追番时间 > 2小时
- 番剧下载成功率 > 95%
- 文件整理准确率 > 90%
- 系统稳定运行时间 > 99%

## 2. 用户分析

### 2.1 目标用户
- **主要用户**：个人番剧爱好者（18-35岁）
- **次要用户**：小型字幕组、番剧分享社区管理员

### 2.2 用户画像

#### 画像1：资深番剧收藏家 - 小李
- **基本信息**：25岁，IT从业者，每季追10+部番
- **使用场景**：希望自动下载高质量资源，建立个人媒体库
- **核心需求**：
  - 支持多个RSS源，灵活的过滤规则
  - 自动选择最佳画质和字幕组
  - 完善的文件命名和整理功能
- **痛点**：
  - 手动检查更新太麻烦
  - 下载的文件命名混乱
  - 错过喜欢的番剧更新

#### 画像2：休闲追番用户 - 小王
- **基本信息**：22岁，大学生，每季追3-5部热门番
- **使用场景**：只想简单追几部热门番，不想太复杂
- **核心需求**：
  - 简单易用的界面
  - 一键订阅热门番剧
  - 及时的更新通知
- **痛点**：
  - 不了解RSS和过滤规则
  - 担心配置太复杂
  - 存储空间有限

### 2.3 用户旅程图

```
发现阶段 → 配置阶段 → 使用阶段 → 管理阶段
   ↓           ↓           ↓           ↓
搜索番剧 → 添加订阅 → 自动下载 → 查看管理
         → 设置规则 → 接收通知 → 整理文件
         → 配置下载 → 追番观看 → 清理空间
```

### 3.2 核心功能详述

#### 3.2.1 订阅管理模块

**功能描述**：管理RSS订阅源，设置过滤规则，实现番剧的自动追踪。

**功能清单**：
| 功能点 | 描述 | 优先级 | 备注 |
|--------|------|--------|------|
| RSS源添加 | 支持添加多个RSS源URL | P0 | 支持主流动漫资源站 |
| RSS源测试 | 验证RSS源可用性 | P0 | 显示最近更新项 |
| 番剧搜索 | 通过关键词搜索番剧 | P0 | 支持中文/日文/罗马音 |
| 快速订阅 | 一键订阅热门番剧 | P1 | 预设热门番剧列表 |
| 过滤规则 | 设置标题、字幕组、画质过滤 | P0 | 支持正则表达式 |
| 订阅分组 | 将订阅分类管理 | P2 | 如"新番"、"旧番" |

**交互流程**：
1. 用户进入订阅管理页面
2. 添加RSS源或搜索番剧
3. 选择要订阅的番剧
4. 设置过滤规则（可选）
5. 确认订阅

#### 3.2.2 资源管理模块

**功能描述**：自动解析资源信息，识别番剧元数据，为下载决策提供支持。

**功能清单**：
| 功能点 | 描述 | 优先级 | 备注 |
|--------|------|--------|------|
| 标题解析 | 解析番名、集数、字幕组等 | P0 | 支持多种命名格式 |
| 画质识别 | 识别720P/1080P/4K等 | P0 | 优先下载高画质 |
| 字幕组识别 | 识别发布字幕组 | P1 | 支持偏好设置 |
| 重复检测 | 避免重复下载 | P0 | 基于文件名和大小 |
| 资源评分 | 根据规则自动评分 | P2 | 综合画质、字幕组等 |

#### 3.2.3 下载管理模块

**功能描述**：与qBittorrent集成，管理下载任务，优化下载体验。

**功能清单**：
| 功能点 | 描述 | 优先级 | 备注 |
|--------|------|--------|------|
| qBittorrent连接 | 配置并连接qBittorrent | P0 | 支持远程连接 |
| 自动添加任务 | 自动将种子添加到下载 | P0 | 可设置下载路径 |
| 队列管理 | 控制同时下载数量 | P1 | 避免带宽拥堵 |
| 下载限速 | 设置上传/下载速度限制 | P1 | 分时段限速 |
| 任务监控 | 实时查看下载进度 | P0 | 显示速度、剩余时间 |
| 失败重试 | 自动重试失败的下载 | P1 | 可设置重试次数 |

#### 3.2.4 文件处理模块

**功能描述**：智能整理下载完成的文件，保持媒体库的整洁有序。

**功能清单**：
| 功能点 | 描述 | 优先级 | 备注 |
|--------|------|--------|------|
| 智能重命名 | 按规则重命名文件 | P0 | 如"番剧名 - 集数" |
| 目录整理 | 自动分类到对应文件夹 | P0 | 按番剧/季度分类 |
| 字幕提取 | 提取内嵌字幕 | P2 | 生成独立字幕文件 |
| 元数据写入 | 写入番剧信息到文件 | P2 | 方便媒体服务器识别 |
| 空间清理 | 自动清理旧文件 | P1 | 可设置保留策略 |

#### 3.2.5 通知系统

**功能描述**：多渠道推送下载状态和更新通知。

**功能清单**：

| 邮件通知 | 发送下载完成邮件 | P1 | 支持自定义模板 |
| Webhook | 支持自定义Webhook | P2 | 集成其他服务 |
| Telegram推送 | 推送到Telegram | P1 | 需要Bot Token |
| 推送规则 | 设置通知触发条件 | P1 | 如仅失败时通知 |

### 3.3 用户故事

#### Story 1: 快速开始追番
**作为**一个新用户  
**我想要**快速订阅热门番剧  
**以便于**马上开始自动下载  

**验收标准**：
- 提供热门番剧推荐列表
- 一键订阅功能
- 默认配置即可使用

#### Story 2: 自定义下载规则
**作为**一个资深用户  
**我想要**设置详细的过滤规则  
**以便于**只下载特定字幕组的特定画质  

**验收标准**：
- 支持正则表达式过滤
- 可以设置多个条件组合
- 规则可以保存和复用

#### Story 3: 媒体库自动整理
**作为**一个收藏爱好者  
**我想要**下载的文件自动整理  
**以便于**保持媒体库的整洁  

**验收标准**：
- 自动重命名为统一格式
- 按番剧名称创建文件夹
- 自动移动到指定目录

## 4. 非功能需求

### 4.1 性能需求
- 页面加载时间 < 2秒
- API响应时间 < 500ms
- 支持同时管理 > 100个订阅
- 支持同时下载 > 10个任务

### 4.2 可用性需求
- 系统可用性 > 99%
- 支持断点续传
- 异常自动恢复
- 数据定期备份

### 4.3 安全需求
- 用户认证和授权
- API访问令牌
- 敏感信息加密存储
- 操作日志审计

### 4.4 兼容性需求
- 浏览器：Chrome、Firefox、Safari最新版本
- qBittorrent：4.3.0及以上版本
- 操作系统：Linux、Windows、macOS
- Docker部署支持

### 4.5 可扩展性需求
- 插件化架构设计
- 支持添加新的下载工具
- 支持添加新的通知渠道
- API开放给第三方调用

## 5. 界面原型说明

### 5.1 主要页面结构
1. **仪表板**：显示系统状态、最近下载、订阅更新
2. **订阅管理**：RSS源列表、订阅列表、添加订阅
3. **下载管理**：当前下载、下载历史、下载设置
4. **媒体库**：已下载番剧列表、文件浏览、批量操作
5. **系统设置**：基础设置、下载设置、通知设置、高级选项

### 5.2 交互原则
- 简洁直观的界面设计
- 关键操作需要二次确认
- 实时反馈操作结果
- 支持批量操作
- 响应式布局适配移动端

## 6. 技术约束

### 6.1 技术栈
- 后端：FastAPI (Python 3.8+)
- 前端：Vue 3 + TypeScript
- 数据库：SQLite（默认）/ PostgreSQL（可选）
- 任务队列：Celery + Redis
- 容器化：Docker + Docker Compose

### 6.2 第三方依赖
- qBittorrent Web API
- RSS解析库
- 番剧信息API（如Bangumi API）
- 通知服务API
